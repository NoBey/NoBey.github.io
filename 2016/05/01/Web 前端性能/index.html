Web 前端性能

2016-05-01

Web 前端性能的两个重要指标 度量页面性能存在着许多指标参数 ，本文涉及 到其中的两个技术指标［２ ］ ：TDD 和 TSR ．
（１）TDD（Time to Document Downloaded）文 档下载完成时间 ：即整个页面中所包含的所有数据 对象如 JS 脚本 、视频图片以及文字等下载完成所 用时间 ．
（２ ） TSR （TimetoStart Render ）开始渲染时 间 ：用户从浏览器发送请求直到页面产生响应并显 示内容的阶段所用时常 ．
１ ．１ 版本协议的 HT TP 规定［３］ ：在同一时 间 ，客户端最多只能向服务器端发出两个连接请 求 ，并且默认所有的连接均为持久连接 ．这一规定 将浏览器的并发连接数限定为二 ．

JavaScript 加载优化
首先分析已经完成的网页，对页面中可以延迟加载的部分进行识别。筛选代 码时需要制定规则进行筛选。在筛选彩购网的代码时，规则为选择页面加载完毕 时，还没有运行的 JavaScript 代码。具体的筛选步骤为， a.记录所有代码中的函数，包括作用域和函数名； b.加载网页，同时记录对可视区域内的元素进行了操作的代码； c.如果被记录的函数在网页加载完毕运行过，则从记录中将其删除

从用户体验出发的几个核心时间指标包括：Start Render、DOM Ready、Page Load、TTI。不同的性能指标对用户体验的影响是不同的，而指标本身受哪些因素的影响也是不同的。一般的前端优化都会考虑前面三个的优化点，而最后的TTI优化往往被忽略。我们却碰到了必须解决这个问题的点，业务场景发生在需要加入购物的按钮上。比如详情页的下单按钮。
